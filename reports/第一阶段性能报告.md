# 第一阶段性能报告

## 测试目标

评估第一阶段基础可运行版的接口响应情况，重点关注单条分析接口、重复文本缓存和 Docker 部署稳定性。

## 测试环境

| 项目 | 内容 |
| --- | --- |
| 操作系统 | Windows |
| 容器环境 | Docker Desktop |
| 后端服务 | FastAPI + Uvicorn |
| 数据库 | MySQL 8.4 |
| 缓存 | Redis 7.4 |
| 模型 | 本地开发支持 BERT 权重推理；Docker 轻量镜像无权重时使用规则兜底 |

## 测试指标

| 指标 | 说明 |
| --- | --- |
| 单条接口响应时间 | `/api/sentiment/single` 平均响应时间 |
| 缓存命中效果 | 相同文本第二次请求是否命中缓存 |
| 批量分析耗时 | 上传 CSV 后完成分析所需时间 |
| 服务稳定性 | Docker Compose 启动后容器是否持续运行 |

## 建议测试方法

### 单条分析

连续请求单条分析接口，记录平均响应时间。

```powershell
curl -X POST http://127.0.0.1:8000/api/sentiment/single `
  -H "Content-Type: application/json" `
  -d "{\"text\":\"物流很快，质量不错，下次还会买\",\"product_id\":\"phone\"}"
```

### 缓存验证

对同一条文本连续请求两次，第二次响应中的 `cached` 字段应为 `true`。

### 批量分析

准备包含 `content` 字段的 CSV 文件，通过 `/api/sentiment/batch` 上传，记录任务返回时间和成功数量。

## 第一阶段结论

第一阶段以“跑通链路”为主，当前已具备 BERT 训练、评估和本地推理能力；Docker 部署侧为了保证一键启动，默认使用轻量 API 镜像，未放置权重时自动降级到规则模型。后续如要评估模型性能，需要在统一机器上分别测试 BERT 推理、缓存命中和批量分析耗时。
