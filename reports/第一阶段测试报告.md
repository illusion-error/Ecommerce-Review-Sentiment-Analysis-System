# 第一阶段测试报告

## 测试目标

验证基础可运行版是否形成闭环，包括后端接口、批量分析、历史查询、统计导出、MySQL 持久化、Redis 缓存和 Docker Compose 启动。

## 测试环境

| 项目 | 内容 |
| --- | --- |
| 操作系统 | Windows |
| 容器环境 | Docker Desktop |
| 后端框架 | FastAPI |
| 数据库 | MySQL 8.4 |
| 缓存 | Redis 7.4 |
| 前端 | Nginx 第一阶段占位页 |

## 测试范围

| 编号 | 测试项 | 接口或对象 | 预期结果 |
| --- | --- | --- | --- |
| T01 | 健康检查 | `GET /api/health` | 返回 `status=ok` |
| T02 | 单条评论分析 | `POST /api/sentiment/single` | 返回情感标签、置信度和记录 ID |
| T03 | 重复文本缓存 | `POST /api/sentiment/single` | 第二次相同文本命中缓存 |
| T04 | 批量文件分析 | `POST /api/sentiment/batch` | 返回任务 ID、成功数量和统计结果 |
| T05 | 批量任务查询 | `GET /api/tasks/{task_id}` | 返回任务完成状态 |
| T06 | 历史记录查询 | `GET /api/history` | 返回分页历史记录 |
| T07 | 统计汇总 | `GET /api/statistics/summary` | 返回正负向数量和缓存统计 |
| T08 | 结果导出 | `GET /api/export/{task_id}` | 返回 CSV 或 Excel 文件 |
| T09 | Docker 启动 | `docker compose up -d --build` | backend、frontend、mysql、redis 均启动 |
