# 性能测试报告

## 测试目标

验证第二阶段系统在新增评论洞察接口后仍然稳定可用，并记录核心接口的平均耗时、P95 耗时和缓存命中效果。

## 测试环境

| 项目 | 内容 |
| --- | --- |
| 操作系统 | Windows |
| 容器环境 | Docker Desktop |
| 后端 | FastAPI + Uvicorn |
| 前端 | Vue + Vite + Nginx |
| 数据库 | MySQL 8.4 |
| 缓存 | Redis 7.4 |
| 后端端口 | `18000` |
| 前端端口 | `15173` |
| 测试数据 | `data/demo/demo_comments.csv`，36 条评论 |

## 测试方法

使用 Docker Compose 启动完整系统：

```powershell
docker compose up -d --build
```

上传演示数据：

```powershell
curl.exe -F "file=@data/demo/demo_comments.csv;type=text/csv" -F "product_id=phone" http://127.0.0.1:18000/api/sentiment/batch
```

接口耗时统计方法：

- 单条分析、高频词、维度评分、商品总结各请求 10 次。
- 批量分析上传 36 条 CSV，请求 3 次。
- 使用 PowerShell `Stopwatch` 记录毫秒耗时。
- P95 使用排序后第 `ceil(n * 0.95)` 个样本。

## 本次实测结果

| 测试项 | 请求次数 | 平均耗时 | P95 耗时 | 最小耗时 | 最大耗时 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 单条分析接口 | 10 | 120.20 ms | 256.12 ms | 61.08 ms | 256.12 ms |
| 高频词接口 | 10 | 42.10 ms | 70.47 ms | 34.79 ms | 70.47 ms |
| 维度评分接口 | 10 | 34.57 ms | 59.82 ms | 29.95 ms | 59.82 ms |
| 商品总结接口 | 10 | 43.77 ms | 72.71 ms | 38.63 ms | 72.71 ms |
| 批量分析接口，36 条 CSV | 3 | 3948.25 ms | 4436.65 ms | 3330.97 ms | 4436.65 ms |

## 缓存验证

重复请求相同文本：

```json
{
  "cached": true
}
```

说明 Redis 缓存链路可用。由于测试前该文本已被请求过，连续验证时第一次和第二次均命中缓存。

## Docker 复验结果

| 服务 | 状态 |
| --- | --- |
| backend | Up，`18000 -> 8000` |
| frontend | Up，`15173 -> 80` |
| mysql | Up，healthy，`3307 -> 3306` |
| redis | Up，`6380 -> 6379` |

后端健康检查：


前端页面：

```text
http://127.0.0.1:15173
```

前端代理健康检查：


三项均已验证通过。

## 结论

第二阶段新增的高频词、维度评分和商品总结接口在 36 条演示数据下响应稳定，平均耗时均在 50 ms 左右。批量分析 36 条评论平均耗时约 3.95 秒，可满足演示和阶段验收需要。后续若接入更大的 BERT 模型或更大规模数据，应继续扩大样本量并补充分位数统计。
