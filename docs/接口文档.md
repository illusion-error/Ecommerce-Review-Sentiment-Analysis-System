# 接口文档

## 基础信息

Docker Compose 启动后，本机访问地址如下：

| 服务 | 地址 |
| --- | --- |
| 后端 API | `http://127.0.0.1:18000` |
| 前端页面 | `http://127.0.0.1:15173` |
| FastAPI 在线文档 | `http://127.0.0.1:18000/docs` |

Docker Compose 内部服务地址：

```text
http://backend:8000
```

## 通用响应格式

```json
{
  "success": true,
  "message": "ok",
  "data": {},
  "code": 0
}
```

## 第一阶段接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/health` | 服务健康检查 |
| POST | `/api/sentiment/single` | 单条评论情感分析 |
| POST | `/api/sentiment/batch` | 上传 CSV/Excel 批量分析 |
| GET | `/api/tasks/{task_id}` | 查询批量任务状态 |
| GET | `/api/history` | 分页查询分析历史 |
| GET | `/api/statistics/summary` | 查询情感统计汇总 |
| GET | `/api/export/{task_id}` | 导出批量分析结果 |

## 第二阶段接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/insights/keywords` | 获取正负向高频词 |
| GET | `/api/insights/aspects` | 获取价格、物流、质量三维度评分 |
| GET | `/api/summary/product` | 获取商品评论规则总结 |

## 单条评论分析

```http
POST /api/sentiment/single
Content-Type: application/json
```

请求体：

```json
{
  "text": "物流很快，质量不错，下次还会买",
  "product_id": "phone",
  "comment_time": "2026-07-13"
}
```

核心响应字段：

| 字段 | 说明 |
| --- | --- |
| `record_id` | 分析记录 ID |
| `text` | 原始评论 |
| `clean_text` | 清洗后的评论 |
| `label` | 标签，1 为正向，0 为负向 |
| `sentiment` | `positive` 或 `negative` |
| `confidence` | 置信度 |
| `strength` | 情感强度 |
| `cached` | 是否命中缓存 |

## 批量分析

```http
POST /api/sentiment/batch
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `file` | 是 | CSV 或 Excel 文件 |
| `product_id` | 否 | 商品或分类 ID |

上传文件建议包含 `content` 字段作为评论文本列。

演示数据上传示例：

```powershell
curl.exe -F "file=@data/demo/demo_comments.csv;type=text/csv" -F "product_id=phone" http://127.0.0.1:18000/api/sentiment/batch
```

## 历史、统计和导出

历史查询：

```text
GET /api/history?page=1&page_size=20&sentiment=positive&product_id=phone&start_time=2026-07-01&end_time=2026-07-13
```

统计汇总：

```text
GET /api/statistics/summary?product_id=phone
```

统计接口返回 `intensity_distribution` 字段，顺序对应 `0-2`、`2-4`、`4-6`、`6-8`、`8-10` 五个情感强度区间。

结果导出：

```text
GET /api/export/{task_id}?file_type=csv
GET /api/export/{task_id}?file_type=xlsx
```

## 高频词接口

```text
GET /api/insights/keywords?product_id=phone&top_k=10
```

查询参数：

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `product_id` | 否 | 商品 ID，不传则统计全部 |
| `top_k` | 否 | 返回词数，默认 30，范围 1-100 |
| `start_time` | 否 | 开始日期 |
| `end_time` | 否 | 结束日期 |

响应核心字段：

| 字段 | 说明 |
| --- | --- |
| `record_count` | 参与统计的记录数 |
| `positive_words` | 正向高频词数组 |
| `negative_words` | 负向高频词数组 |

词条结构：

```json
{
  "word": "质量",
  "count": 24,
  "weight": 1.0
}
```

## 维度评分接口

```text
GET /api/insights/aspects?product_id=phone
```

响应核心字段：

| 字段 | 说明 |
| --- | --- |
| `record_count` | 参与统计的记录数 |
| `aspects` | 价格、物流、质量维度数组 |

维度结构：

```json
{
  "name": "价格",
  "score": 86.67,
  "total_count": 15,
  "positive_count": 13,
  "negative_count": 2,
  "keywords": ["价格", "优惠", "便宜"],
  "status": "ok"
}
```

无数据时：

```json
{
  "score": 0.0,
  "total_count": 0,
  "status": "no_data"
}
```

## 商品总结接口

```text
GET /api/summary/product?product_id=phone
```

响应核心字段：

| 字段 | 说明 |
| --- | --- |
| `mode` | 当前为 `rule_fallback` |
| `advantages` | 优点列表 |
| `disadvantages` | 缺点列表 |
| `buying_advice` | 购买建议 |
| `evidence` | 生成总结所依据的关键词和维度分数 |

## 前端联调说明

前端生产容器由 Nginx 托管 Vue 构建结果，并将 `/api/` 代理到后端容器：

```text
frontend /api/* -> backend:8000/api/*
```

因此前端页面中请求 `/api/health` 会自动访问后端服务。
