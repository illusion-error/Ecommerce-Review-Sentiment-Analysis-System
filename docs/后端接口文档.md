# 后端接口文档

## 1. 基础信息

- 默认地址：`http://127.0.0.1:8000`
- 框架：FastAPI
- 返回格式：

```json
{
  "success": true,
  "message": "ok",
  "data": {},
  "code": 0
}
```

## 2. 健康检查

```http
GET /api/health
```

返回后端版本、数据库模式和缓存状态。

## 3. 单条评论情感分析

```http
POST /api/sentiment/single
Content-Type: application/json
```

请求：

```json
{
  "text": "物流很快，质量也不错，下次还会买",
  "product_id": "phone",
  "comment_time": "2026-07-07"
}
```

返回：

```json
{
  "success": true,
  "message": "分析成功",
  "data": {
    "record_id": 1,
    "text": "物流很快，质量也不错，下次还会买",
    "clean_text": "物流很快质量也不错下次还会买",
    "label": 1,
    "sentiment": "positive",
    "confidence": 0.79,
    "strength": 7.9,
    "cached": false,
    "product_id": "phone"
  },
  "code": 0
}
```

## 4. 批量评论分析

```http
POST /api/sentiment/batch
Content-Type: multipart/form-data
```

字段：

- `file`：CSV/Excel 文件。
- `product_id`：可选。

文件文本列优先识别：`content`、`comment`、`text`、`raw_text`、`评论`、`评价`、`评论内容`。如果都不存在，默认使用第一列。

返回：

```json
{
  "success": true,
  "message": "批量分析完成",
  "data": {
    "task_id": "batch_xxx",
    "total": 2,
    "success_count": 2,
    "failed_count": 0,
    "positive_count": 1,
    "negative_count": 1,
    "avg_strength": 4.8,
    "results": []
  },
  "code": 0
}
```

## 5. 批量任务状态

```http
GET /api/tasks/{task_id}
```

返回任务状态、总行数、成功行数、失败行数、正负向数量和平均强度。

## 6. 历史查询

```http
GET /api/history?page=1&page_size=20&sentiment=positive&product_id=phone&start_time=2026-07-01&end_time=2026-07-09
```

参数：

- `page`：页码，从 1 开始。
- `page_size`：每页数量，1-100。
- `sentiment`：可选，`positive` 或 `negative`。
- `product_id`：可选。
- `start_time`：可选，开始日期，建议格式 `YYYY-MM-DD`。
- `end_time`：可选，结束日期，建议格式 `YYYY-MM-DD`。

## 7. 统计汇总

```http
GET /api/statistics/summary
GET /api/statistics/summary?product_id=phone
```

返回：

- `total`
- `positive_count`
- `negative_count`
- `positive_ratio`
- `negative_ratio`
- `avg_strength`
- `intensity_distribution`
- `cache`

`intensity_distribution` 顺序对应 `0-2`、`2-4`、`4-6`、`6-8`、`8-10` 五个情感强度区间。

## 8. 结果导出

```http
GET /api/export/{task_id}?file_type=csv
GET /api/export/{task_id}?file_type=xlsx
```

用于下载批量任务的分析结果。

## 9. 高频词洞察

```http
GET /api/insights/keywords?product_id=phone&top_k=30&start_time=2026-07-01&end_time=2026-07-13
```

参数：

- `product_id`：可选，商品或品类 ID；为空时统计全部历史记录。
- `top_k`：可选，返回词数量，范围 1-100，默认 30。
- `start_time` / `end_time`：可选，按分析记录创建时间筛选。

返回：

```json
{
  "success": true,
  "message": "关键词统计成功",
  "data": {
    "product_id": "phone",
    "top_k": 30,
    "record_count": 20,
    "positive_words": [
      {"word": "物流", "count": 5, "weight": 1.0}
    ],
    "negative_words": [
      {"word": "破损", "count": 3, "weight": 1.0}
    ]
  },
  "code": 0
}
```

说明：空数据时 `positive_words` 和 `negative_words` 返回空数组，不返回 500。

## 10. 评价维度雷达图

```http
GET /api/insights/aspects?product_id=phone
```

返回价格、物流、质量三个维度的评分和命中数量：

```json
{
  "success": true,
  "message": "维度评分成功",
  "data": {
    "product_id": "phone",
    "record_count": 20,
    "aspects": [
      {
        "name": "价格",
        "score": 75.0,
        "total_count": 4,
        "positive_count": 3,
        "negative_count": 1,
        "keywords": ["价格", "划算"],
        "status": "ok"
      }
    ]
  },
  "code": 0
}
```

评分规则：`score = positive_count / total_count * 100`。没有命中评论时返回 `score=0`、`status=no_data`。

## 11. 商品评论总结

```http
GET /api/summary/product?product_id=phone
```

返回基于历史评论的规则兜底总结：

```json
{
  "success": true,
  "message": "总结生成成功",
  "data": {
    "product_id": "phone",
    "record_count": 20,
    "mode": "rule_fallback",
    "advantages": ["正向评论中高频出现：物流、质量、划算。"],
    "disadvantages": ["负向评论中高频出现：破损、太贵。"],
    "buying_advice": "该商品更适合重视物流体验的用户；如果非常关注质量，建议结合负向评论进一步判断。",
    "evidence": {
      "positive_words": ["物流", "质量", "划算"],
      "negative_words": ["破损", "太贵"],
      "aspect_scores": {"价格": 75.0, "物流": 83.33, "质量": 60.0}
    }
  },
  "code": 0
}
```

该接口不强依赖 LLM。没有外部 API Key 时仍返回 `rule_fallback` 结果，内容只来自词云和维度评分证据。
