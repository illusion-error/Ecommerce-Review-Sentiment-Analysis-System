# 电商评论情感分析系统四人分工与两阶段开发实施文档

## 1. 项目定位

本项目是一个面向电商评论的智能文本情感分析系统。系统输入商品评论文本或评论文件，输出正向/负向情感类别、置信度、情感强度分值，并在前端展示情感分布、趋势统计和历史分析结果。

技术路线：

- 模型：`bert-base-chinese` 微调 + CPU 量化推理。
- 后端：FastAPI。
- 前端：Vue3 + Element Plus + ECharts。
- 数据库：MySQL。
- 缓存：Redis。
- 部署：Docker Compose。

## 2. 两阶段建设目标

### 2.1 第一阶段：基础可运行版

目标：先做出一条完整闭环，让系统可以从数据、模型、接口、页面到部署全部跑通。

必须完成：

1. 数据清洗脚本可运行，能输出 `cleaned_full_data.csv`、`train.csv`、`val.csv`、`test.csv`。
2. BERT 微调脚本可运行，能保存模型权重和评估结果。
3. 后端提供单条分析、批量分析、历史查询、导出接口。
4. 前端提供文本输入、文件上传、结果表格、统计图表。
5. MySQL 能保存分析记录，Redis 可做重复文本缓存。
6. Docker Compose 能启动前端、后端、MySQL、Redis。
7. 输出接口文档、测试报告、模型训练报告。

### 2.2 第二阶段：优化展示版

目标：在基础功能跑通后，增强模型性能、系统稳定性和展示效果。

建议完成：

1. 模型量化与 CPU 推理耗时对比。
2. 训练过程可视化，包括 loss、accuracy、F1 曲线。
3. 增加情感强度 0-10 分计算逻辑。
4. 增加批量任务状态轮询和错误行报告。
5. 增加 Redis 命中率统计。
6. 增加压力测试、接口响应时间统计和缓存优化报告。
7. 增加前端 BI 看板，包括正负面分布、时间趋势、商品类别对比。
8. 增加一键部署说明和演示数据。

## 5. 后端接口字段约定

### 5.1 单条情感分析

请求：

```http
POST /api/sentiment/single
Content-Type: application/json
```

```json
{
  "text": "商品质量很好，物流也很快",
  "product_id": "phone"
}
```

响应：

```json
{
  "success": true,
  "message": "分析成功",
  "data": {
    "text": "商品质量很好，物流也很快",
    "clean_text": "商品质量很好物流也很快",
    "label": 1,
    "sentiment": "positive",
    "confidence": 0.93,
    "strength": 9.3,
    "cached": false
  }
}
```

### 5.2 批量文件分析

请求：

```http
POST /api/sentiment/batch
Content-Type: multipart/form-data
```

字段：

- `file`：CSV/Excel 文件。
- `product_id`：可选商品类别。

响应：

```json
{
  "success": true,
  "data": {
    "task_id": "batch_20260707_001",
    "total": 100,
    "positive_count": 72,
    "negative_count": 28,
    "avg_strength": 7.4
  }
}
```

### 5.3 历史查询

```http
GET /api/history?page=1&page_size=20&sentiment=positive
```

返回字段：

- `items`：当前页记录。
- `total`：总记录数。
- `page`：当前页。
- `page_size`：每页数量。

## 6. 数据库表建议

### 6.1 analysis_records

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT PK | 主键 |
| `task_id` | VARCHAR(64) | 批量任务 ID，单条任务可为空 |
| `product_id` | VARCHAR(64) | 商品类别 |
| `raw_text` | TEXT | 原始评论 |
| `clean_text` | TEXT | 清洗后文本 |
| `label` | TINYINT | 0 负向，1 正向 |
| `sentiment` | VARCHAR(16) | positive/negative |
| `confidence` | FLOAT | 模型置信度 |
| `strength` | FLOAT | 0-10 情感强度 |
| `cached` | BOOLEAN | 是否来自缓存 |
| `created_at` | DATETIME | 创建时间 |

### 6.2 batch_tasks

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `task_id` | VARCHAR(64) PK | 批量任务 ID |
| `filename` | VARCHAR(255) | 文件名 |
| `status` | VARCHAR(32) | running/completed/failed |
| `total` | INT | 总行数 |
| `success_count` | INT | 成功行数 |
| `failed_count` | INT | 失败行数 |
| `positive_count` | INT | 正向数量 |
| `negative_count` | INT | 负向数量 |
| `avg_strength` | FLOAT | 平均强度 |
| `created_at` | DATETIME | 创建时间 |
| `finished_at` | DATETIME | 完成时间 |
