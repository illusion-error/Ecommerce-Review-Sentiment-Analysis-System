# 测试用例清单

## 第二阶段测试范围

第二阶段在第一阶段“情感正负向判断”的基础上，新增评论洞察能力，重点覆盖：

- 正负向高频词接口和词云展示数据
- 价格、物流、质量三维度雷达图数据
- 商品评论规则总结接口
- 第一阶段单条分析、批量分析、历史查询、统计导出回归
- Docker Compose 前端、后端、MySQL、Redis 一键部署

## 功能测试用例

| 编号 | 模块 | 测试内容 | 操作 | 预期结果 |
| --- | --- | --- | --- | --- |
| TC-D2-01 | Docker | Compose 配置校验 | 执行 `docker compose config` | 配置无语法错误 |
| TC-D2-02 | Docker | 一键启动服务 | 执行 `docker compose up -d --build` | backend、frontend、mysql、redis 均为 Up，MySQL 为 healthy |
| TC-D2-03 | 前端 | Vue 生产构建 | 在 `frontend` 执行 `npm.cmd run build` | 构建成功并生成 `dist` |
| TC-D2-04 | 前端 | 页面访问 | 访问 `http://127.0.0.1:15173` | 页面返回 200 |
| TC-D2-05 | 后端 | 健康检查 | 请求 `GET /api/health` | 返回 `status=ok` |
| TC-D2-06 | 后端 | 单条分析回归 | 请求 `POST /api/sentiment/single` | 返回 label、sentiment、confidence、cached |
| TC-D2-07 | 缓存 | 重复文本缓存 | 连续两次分析相同文本 | 第二次 `cached=true` |
| TC-D2-08 | 后端 | 批量分析回归 | 上传含 `content` 字段的 CSV | 返回 task_id、success_count、positive_count、negative_count |
| TC-D2-09 | 后端 | 历史查询回归 | 请求 `GET /api/history` | 支持分页、情感筛选、商品筛选、时间筛选 |
| TC-D2-10 | 后端 | 统计汇总回归 | 请求 `GET /api/statistics/summary` | 返回正负向统计和 `intensity_distribution` |
| TC-D2-11 | 后端 | 导出回归 | 请求 `GET /api/export/{task_id}` | 可导出 CSV 或 XLSX |
| TC-D2-12 | 洞察 | 高频词接口 | 请求 `GET /api/insights/keywords?product_id=phone&top_k=10` | 返回 `positive_words`、`negative_words`、`record_count` |
| TC-D2-13 | 洞察 | 高频词空数据 | 使用不存在的 product_id 请求高频词接口 | 返回空数组，不返回 500 |
| TC-D2-14 | 洞察 | 维度评分接口 | 请求 `GET /api/insights/aspects?product_id=phone` | 返回价格、物流、质量三维度，分数范围 0-100 |
| TC-D2-15 | 洞察 | 维度评分空数据 | 使用不存在的 product_id 请求维度接口 | 三个维度返回 `status=no_data`、`score=0` |
| TC-D2-16 | 洞察 | 商品总结接口 | 请求 `GET /api/summary/product?product_id=phone` | 返回 advantages、disadvantages、buying_advice、evidence |
| TC-D2-17 | 前后端联调 | 前端代理后端接口 | 请求 `http://127.0.0.1:15173/api/health` | 通过 Nginx 代理返回后端健康检查结果 |
| TC-D2-18 | 前端真实数据 | 打开前端看板、单条分析和历史记录 | 页面数据来自真实 API，不出现固定 mock 数据或强制演示模式 |

## 自动化测试命令

Docker 环境后端测试：

```powershell
docker compose run --rm --no-deps -e DATABASE_BACKEND=sqlite backend python -m pytest -q
```

前端构建测试：

```powershell
cd frontend
npm.cmd run build
```

Docker 部署复验：

```powershell
docker compose up -d --build
docker compose ps
```
